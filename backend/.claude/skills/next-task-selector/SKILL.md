# Next Task Selector

로드맵에서 다음에 진행할 태스크를 추천한다.

## 선택 기준

### 1. 의존성 (Dependency)
선행 태스크가 완료되어야 시작 가능

### 2. 우선순위 (Priority)
- Critical Path 태스크 우선
- 블로커 제거 우선

### 3. 연속성 (Continuity)
- 같은 영역(파일/모듈) 태스크 연속 처리
- 컨텍스트 스위칭 최소화

### 4. 크기 (Size)
- 남은 시간에 맞는 크기
- 작은 태스크로 모멘텀 유지

## 워크플로우

### Step 1: 로드맵 상태 파악
- 전체 진행률
- 완료된 태스크
- 진행 중인 태스크
- 대기 중인 태스크

### Step 2: 가용 태스크 필터링
- 의존성 충족된 태스크만
- 블로커 없는 태스크

### Step 3: 우선순위 정렬
선택 기준 적용

### Step 4: 추천

## 출력 형식

### 다음 태스크 추천
```markdown
# 🎯 다음 태스크 추천

**현재 상태**: Phase A 진행 중 (50%)

---

## 추천 태스크

### 1순위: A-3. 에러 응답 표준화
**이유**: 
- ✅ 의존성 충족 (A-1, A-2 완료)
- 🔥 다음 Phase 블로커
- ⏱️ 예상 시간: 2시간

**시작하기**:
```
1. app/exceptions/ 디렉토리 생성
2. 베이스 예외 클래스 정의
3. 글로벌 핸들러 등록
```

---

### 2순위: A-4. Postman Collection 생성
**이유**:
- ✅ 의존성 충족
- 📄 문서화 태스크 (부담 적음)
- ⏱️ 예상 시간: 1시간

---

## 건너뛸 태스크

### ⏸️ B-1. 테스트 커버리지 설정
**이유**: A Phase 완료 후 진행 권장

### ⚠️ A-5. API 버저닝
**이유**: A-3 완료 후 가능 (의존성)

---

## 현재 진행 상황

| Phase | 진행률 | 상태 |
|-------|--------|------|
| Phase A | 50% | 🔄 진행중 |
| Phase B | 0% | ⬜ 대기 |
| Phase C | 0% | ⬜ 대기 |

**예상 완료**: Phase A 완료까지 약 4시간
```

## 의존성 분석

### 의존성 그래프 예시
```
A-1 ─────┬──► A-3 ──► A-5
         │
A-2 ─────┘

B-1 (독립)

C-1 ──► C-2 ──► C-3
```

### 병렬 가능 태스크
의존성이 없어 동시 진행 가능:
```markdown
**병렬 가능**:
- A-3 (에러 표준화)
- B-1 (테스트 설정)

같은 시간에 두 명이 나눠서 진행 가능
```

## 컨텍스트 기반 추천

### 최근 작업 기반
```markdown
**최근 완료**: A-2 (Postman Collection)
**연관 태스크**: A-3 (같은 API 문서화 영역)

→ A-3 추천 (컨텍스트 유지)
```

### 시간 기반
```markdown
**남은 시간**: 1시간
**추천**: 작은 태스크 우선

→ A-4 (1시간) 추천
→ A-3 (2시간) 보다 A-4 먼저
```

## 특수 상황

### 블로커 발생 시
```markdown
⚠️ **블로커 감지**

A-3이 외부 라이브러리 이슈로 블로킹됨

**대안 추천**:
1. B-1 (독립 태스크, 블로커 무관)
2. A-4 (A-3 의존 없음)

A-3 블로커 해결 후 복귀
```

### Phase 전환 시
```markdown
🎉 **Phase A 완료!**

**다음 Phase**: Phase B (테스트 강화)

**첫 태스크 추천**: B-1. 테스트 커버리지 설정
- Phase B의 기반 태스크
- 다른 B 태스크의 선행 조건
```

### 작업 재개 시
```markdown
# 📍 현재 위치

**마지막 작업**: 2일 전, A-2 완료

**진행 상황**:
- Phase A: 50% (2/4 완료)
- 마지막 커밋: "feat(api): add Postman collection"

**이어서 할 태스크**: A-3. 에러 응답 표준화

**컨텍스트 복구**:
1. A-1, A-2 변경사항 확인
2. A-3 구현 계획 검토
3. 관련 파일 열기
```

## 빠른 선택 가이드

| 상황 | 추천 기준 |
|------|----------|
| 시간 여유 있음 | Critical Path 태스크 |
| 시간 부족 | 작은 태스크 (Quick Win) |
| 집중 필요 | 독립적 태스크 |
| 컨텍스트 유지 | 같은 영역 태스크 |
| 막힘 | 다른 영역으로 전환 |
