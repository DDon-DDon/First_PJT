# 분석 방법론 워크플로우

데이터 기반 기획의 정당성을 확보하기 위한 3축 분석 전략입니다.

## 1. 수요 신호 분석 (Market Demand)

```bash
# Google Trends 데이터 수집 준비
echo "수요 신호 분석 시작..."
echo "도구: Google Trends, Google Ads Keyword Planner"
echo "기법: 지역/기간별 관심도 변화 모니터링"
echo "핵심: CPC(입찰가)로 상업적 가치 판별"
```

### 분석 포인트
- 지역/기간별 관심도 변화 모니터링
- 키워드 플래너로 CPC 확인하여 상업적 가치 판별
- "언제부터 트래픽이 상승했는지" 확인

## 2. 불만 신호 분석 (VOC/Review)

```bash
# 리뷰 및 커뮤니티 데이터 수집 준비
echo "불만 신호 분석 시작..."
echo "소스: G2, Capterra, 앱스토어 리뷰, Reddit"
echo "기법: 텍스트 마이닝 및 감성 분석"
```

### 분석 포인트
- 텍스트 마이닝으로 반복되는 '불만 키워드' 구조화
- 기존 재고관리 소프트웨어의 불만 사항 역추적
- 카메라 기반 '물리적 자동화' 필요성 도출

## 3. 공급 및 상황 분석 (Trend Report)

```bash
# 트렌드 리포트 분석 준비
echo "공급 신호 분석 시작..."
echo "소스: Gartner, McKinsey, CB Insights"
echo "주요 키워드: Agentic AI, Edge Computing"
```

## 4. 1주 집중 검증 템플릿

| 일차 | 주요 활동 | 결과물 |
|---|---|---|
| **1~2일** | 문제 후보 선정 및 수요 검증 | 상승 키워드 10개 및 검색량 리포트 |
| **3~4일** | 대체 VOC 수집 및 페인포인트 추출 | 반복 불만 Top N 리스트 및 인용구 |
| **5일** | 기술적 타당성 및 리스크 분석 | 기술 청사진 및 리스크 레지스터 |
| **6~7일** | 기획 산출물 통합 및 PRD 작성 | 최종 기획안 (해결안 및 KPI 포함) |
