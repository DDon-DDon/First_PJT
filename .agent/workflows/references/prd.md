# PRD (Product Requirements Document)

## 프로젝트 개요

**제품명**: 똔똔 (DoneDone)
**목표**: 오프라인 매장의 재고 관리 효율화
**타겟 사용자**: 중소 리테일 매장 (화장품, 편의점 등)

## 사용자 역할

| 역할   | 설명        | 권한                                         |
| ------ | ----------- | -------------------------------------------- |
| WORKER | 현장 작업자 | 입고/출고 처리, 재고 조회 (배정된 매장만)    |
| ADMIN  | 관리자      | 모든 기능 + 제품 등록 + 대시보드 + 알림 수신 |

---

## 기능 목록

### P0 (MVP 필수)

#### 1. 재고 입고 기능

**설명**: 바코드 스캔으로 제품 입고 처리

**Acceptance Criteria**:

- [ ] 바코드 스캔/입력 시 1초 이내 제품 정보 표시
- [ ] 스캔 불가 시 수동 바코드 입력 가능
- [ ] 수량 입력 후 저장 시 재고 즉시 반영
- [ ] 입고 완료 시 성공 메시지 표시
- [ ] 오프라인에서도 입고 처리 가능 (로컬 저장)

**관련 엔티티**: Product, InventoryTransaction, CurrentStock

---

#### 2. 재고 출고 기능

**설명**: 바코드 스캔으로 제품 출고 처리

**Acceptance Criteria**:

- [ ] 바코드 스캔/입력 시 1초 이내 제품 정보 표시
- [ ] 현재 재고 수량 실시간 표시
- [ ] 재고 부족 시 출고 차단 및 경고 메시지
- [ ] 출고 완료 후 안전재고 이하 시 관리자 알림
- [ ] 오프라인에서도 출고 처리 가능 (재고 체크 로컬)

**관련 엔티티**: Product, InventoryTransaction, CurrentStock

---

#### 3. 신규 제품 등록

**설명**: 새로운 제품을 시스템에 등록

**Acceptance Criteria**:

- [ ] 바코드 스캔 또는 수동 입력으로 바코드 등록
- [ ] 바코드 없는 제품은 내부 바코드 자동 생성 (DON-{카테고리}-{순번})
- [ ] 필수 입력: 바코드, 제품명, 카테고리
- [ ] 선택 입력: 안전재고, 이미지, 메모
- [ ] 중복 바코드 등록 방지

**관련 엔티티**: Product, Category

---

#### 4. 오프라인 동기화

**설명**: 네트워크 없이도 동작하고 복구 시 자동 동기화

**Acceptance Criteria**:

- [ ] 오프라인 상태에서 입출고 처리 가능
- [ ] 오프라인 상태 표시 (인디케이터)
- [ ] 동기화 대기 건수 표시
- [ ] 네트워크 복구 시 자동 동기화 시작
- [ ] 동기화 실패 시 재시도 (최대 3회)
- [ ] 동기화 완료/실패 알림

**관련 엔티티**: InventoryTransaction (synced_at 필드)

---

### P1 (후속 개발)

#### 5. 재고 조정 기능

**설명**: 폐기, 파손 등 재고 수동 조정

**Acceptance Criteria**:

- [ ] 조정 사유 필수 선택 (유통기한만료, 파손, 오류정정, 기타)
- [ ] 조정 수량 입력 (증가/감소)
- [ ] 조정 이력 기록 (누가, 언제, 왜)

**관련 엔티티**: InventoryTransaction (type: ADJUST, reason 필드)

---

#### 6. 재고 현황 대시보드

**설명**: 관리자용 재고 현황 조회

**Acceptance Criteria**:

- [ ] 전체 재고 목록 표시
- [ ] 매장별 필터링
- [ ] 카테고리별 필터링
- [ ] 안전재고 이하 제품 하이라이트
- [ ] 제품별 입출고 타임라인 조회
- [ ] 안전재고 이하 목록 엑셀 내보내기

**관련 엔티티**: Product, CurrentStock, InventoryTransaction

---

## 비기능 요구사항

### 성능

- 바코드 조회 → 제품 조회: < 1초
- 입출고 처리 응답: < 500ms
- 대시보드 로딩: < 2초

### 보안

- JWT 기반 인증
- HTTPS 필수
- 비밀번호 해싱 (bcrypt)

### 가용성

- 오프라인 동작 지원
- 모바일 최적화 (반응형)
